## Phase 3 — Media service — 2026-05-08
Added `App\Services\MediaService` for reusable polymorphic media syncing. It stores uploaded files on the configured disk under `media/{model-type}/{model-id}/{ulid}.{ext}`, updates existing media metadata, deletes removed rows plus files and translations, syncs DE/AR/EN `alt_text` translations, preserves omitted existing rows, and guarantees one primary media item when media remains. Added `tests/Feature/MediaServiceTest.php` with `Storage::fake('public')` covering create, update, delete, primary enforcement, translation sync, and unsaved-model rejection. Carry-over: Product CRUD will be the first controller to consume the service.
