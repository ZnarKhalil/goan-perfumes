# Hetzner Production Server Setup

This guide prepares a fresh Hetzner Linux server for the Goan Perfume Laravel/Inertia website.

Assumptions:

- Server OS: Ubuntu 26.04 LTS.
- You have root SSH access.
- Domain DNS can be pointed to the server before enabling HTTPS.
- App path: `/var/www/goan-perfumes`.
- Deploy user: `znar`.
- PHP target: Ubuntu-packaged PHP, discovered after install. PHP 8.5 is preferred when Ubuntu ships it, but do not use an unsupported PPA on Ubuntu 26.04.
- Database target: PostgreSQL 18.x. Install the latest PostgreSQL 18 patch available from apt instead of pinning exactly `18.1`, because patch releases contain security and bug fixes.
- Web server: Caddy.

Important safety rule: sections 4 and 5 are run as `root`. Do not disable root SSH until `znar` can log in over SSH and `sudo -n true` succeeds. If `sudo` asks `znar` for a password, stop and fix `/etc/sudoers.d/znar` before continuing. After root SSH is disabled, section 6 onward is run as `znar` with `sudo` for system administration commands.

## 1. Why Caddy Instead Of Nginx

Use Caddy for this site.

Reasons:

- Automatic HTTPS and certificate renewal are built in.
- The Caddyfile is smaller and harder to misconfigure than an equivalent Nginx Laravel config.
- It supports PHP-FPM, static file serving, compression, HTTP/2, and HTTP/3 well.
- It is enough for a single Laravel app on one Hetzner VPS.

Use Nginx only if you later need a setup that your ops tooling already standardizes on, very custom reverse proxy behavior, or advanced caching/load-balancing rules. For this project, Caddy is the lower-maintenance choice.

## 3. DNS First

In your DNS provider, create:

```text
A      goanperfume.de     SERVER_IPV4
CNAME  www               goanperfume.de
AAAA   goanperfume.de     SERVER_IPV6   optional
```

Wait until the domain resolves:

```bash
dig +short goanperfume.de
dig +short www.goanperfume.de
```

Caddy cannot issue a trusted certificate until DNS points to the server and ports 80/443 are reachable.

### GoDaddy DNS Setup

Because the domain was bought through GoDaddy, keep DNS simple: leave the domain on GoDaddy nameservers and edit the DNS records there. Do not use GoDaddy domain forwarding, GoDaddy hosting, or a GoDaddy SSL certificate for this Laravel server.

In GoDaddy:

1. Sign in to GoDaddy.
2. Open **My Products**.
3. Find the domain.
4. Select **DNS** or **Manage DNS**.
5. Go to **DNS Records**.

Add or edit these records:

```text
Type   Name   Value           TTL
A      @      SERVER_IPV4     1 hour
CNAME  www    @               1 hour
```

If Hetzner gave the server an IPv6 address and you plan to use IPv6, also add:

```text
Type   Name   Value           TTL
AAAA   @      SERVER_IPV6     1 hour
```

Recommended setup:

- Use `A @ -> SERVER_IPV4` for the root domain.
- Use `CNAME www -> @` so `www.goanperfume.de` follows the root domain.
- Keep GoDaddy default nameservers unless you intentionally move DNS to Hetzner DNS.
- Delete or replace old parked-domain records that point to GoDaddy parking, website builder, or another host.
- Do not delete MX, TXT, SPF, DKIM, DMARC, or email-related records unless you are intentionally changing email.

If GoDaddy already has an `A` record for `@`, edit it instead of creating a duplicate. If GoDaddy already has a `CNAME` or `A` record for `www`, edit it to match the table above.

After saving, wait a few minutes and check from your machine:

```bash
dig +short goanperfume.de
dig +short www.goanperfume.de
```

Expected:

- `goanperfume.de` returns the Hetzner IPv4 address.
- `www.goanperfume.de` resolves to the same target through the `CNAME`.

DNS can work within minutes, but GoDaddy notes that DNS changes can take up to 48 hours globally. Do not start troubleshooting Caddy certificates until `dig` returns the Hetzner IP from outside the server.

## 5. Create A Deploy User

Create the user without a password and add it to `sudo`:

```bash
adduser --disabled-password --gecos "" znar
usermod -aG sudo znar
```

Because `znar` has no password, configure passwordless sudo before disabling root SSH:

```bash
echo 'znar ALL=(ALL) NOPASSWD:ALL' > /etc/sudoers.d/znar
chmod 440 /etc/sudoers.d/znar
visudo -cf /etc/sudoers.d/znar
```

Copy your SSH key from root to the new user:

```bash
rsync --archive --chown=znar:znar /root/.ssh /home/znar
```

Test a second SSH session before closing the root session:

```bash
ssh znar@SERVER_IP
```

In the `znar` SSH session, confirm sudo works without a password:

```bash
sudo -n true
sudo whoami
```

Expected output from `sudo whoami`:

```text
root
```

If either command asks for a password or fails, do not disable root SSH. Go back to the root session and re-check `/etc/sudoers.d/znar`.

Only after the `znar` user works and passwordless sudo has been verified, lock down SSH from the still-open root session:

```bash
nano /etc/ssh/sshd_config
```

Use these values:

```text
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
```

Reload SSH:

```bash
systemctl reload ssh
```

Keep the current root shell open until you confirm a new `znar` SSH login works and `sudo -n true` succeeds. After SSH is reloaded, open or keep a `znar` SSH session and continue from section 6 as `znar`.

## 6. Firewall And Basic Protection

From here onward, run commands from the `znar` SSH session. Use `sudo` for system-level commands.

Allow SSH, HTTP, and HTTPS:

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 443/udp
sudo ufw enable
sudo ufw status verbose
```

`443/udp` is required for HTTP/3/QUIC. Without it, Caddy still serves the site over HTTPS, but browsers fall back to HTTP/2.

Enable fail2ban:

```bash
sudo systemctl enable --now fail2ban
sudo systemctl status fail2ban
```

Make the SSH jail explicit:

```bash
sudo nano /etc/fail2ban/jail.local
```

Use:

```ini
[sshd]
enabled = true
port = ssh
logpath = %(sshd_log)s
backend = systemd
maxretry = 5
findtime = 10m
bantime = 1h
```

Restart fail2ban:

```bash
sudo systemctl restart fail2ban
sudo fail2ban-client status sshd
```

Enable automatic security updates:

```bash
sudo apt install -y unattended-upgrades
sudo dpkg-reconfigure unattended-upgrades
```

Choose `Yes`.

Create a 2 GB swapfile before building frontend assets on the server. This prevents common out-of-memory failures on small Hetzner instances during `npm ci` and `npm run build`.

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
sudo cp /etc/fstab /etc/fstab.bak
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
free -h
```

## 7. Install PHP And Extensions

Continue as `znar`:

```bash
ssh znar@SERVER_IP
```

Ubuntu 26.04 uses the `resolute` codename. Do not use the Ondrej PHP PPA unless it publishes packages for `resolute`; otherwise `apt update` fails with `does not have a Release file`.

If you already added the broken PPA, remove it first:

```bash
sudo add-apt-repository --remove ppa:ondrej/php -y || true
sudo rm -f /etc/apt/sources.list.d/ondrej-ubuntu-php-resolute.sources
sudo rm -f /etc/apt/sources.list.d/ondrej-ubuntu-php-resolute.list
sudo apt update
```

Install PHP-FPM, CLI, and extensions used by Laravel, PostgreSQL, uploads, image processing, and common Composer packages:

```bash
sudo apt install -y \
  php-cli \
  php-fpm \
  php-common \
  php-pgsql \
  php-mbstring \
  php-xml \
  php-curl \
  php-zip \
  php-bcmath \
  php-intl \
  php-gd \
  php-readline
```

Check which PHP version Ubuntu installed:

```bash
php -v
PHP_VERSION="$(php -r 'echo PHP_MAJOR_VERSION.".".PHP_MINOR_VERSION;')"
echo "$PHP_VERSION"
php -m | grep -i opcache
```

If `php -m | grep -i opcache` prints `Zend OPcache`, OPcache is installed. On Ubuntu 26.04 it may be included by the base PHP packages rather than provided as a separate `php-opcache` package.

Tune PHP for production:

```bash
PHP_VERSION="$(php -r 'echo PHP_MAJOR_VERSION.".".PHP_MINOR_VERSION;')"
sudo nano "/etc/php/${PHP_VERSION}/fpm/conf.d/99-goan-perfume.ini"
```

Use:

```ini
upload_max_filesize=20M
post_max_size=24M
memory_limit=256M
max_execution_time=60
max_input_vars=3000
expose_php=Off

opcache.enable=1
opcache.enable_cli=1
opcache.memory_consumption=192
opcache.interned_strings_buffer=16
opcache.max_accelerated_files=20000
opcache.validate_timestamps=0
opcache.save_comments=1
```

Configure PHP-FPM to listen on a stable local TCP port. This avoids hardcoding a versioned socket path in Caddy:

```bash
PHP_VERSION="$(php -r 'echo PHP_MAJOR_VERSION.".".PHP_MINOR_VERSION;')"
sudo cp "/etc/php/${PHP_VERSION}/fpm/pool.d/www.conf" "/etc/php/${PHP_VERSION}/fpm/pool.d/www.conf.bak"
sudo sed -i 's#^listen = .*#listen = 127.0.0.1:9000#' "/etc/php/${PHP_VERSION}/fpm/pool.d/www.conf"
sudo systemctl restart "php${PHP_VERSION}-fpm"
sudo systemctl enable "php${PHP_VERSION}-fpm"
sudo systemctl status "php${PHP_VERSION}-fpm"
```

Because `opcache.validate_timestamps=0`, reload PHP-FPM on every deploy. Use the detected service name:

```bash
PHP_VERSION="$(php -r 'echo PHP_MAJOR_VERSION.".".PHP_MINOR_VERSION;')"
sudo systemctl reload "php${PHP_VERSION}-fpm"
```

The app currently requires `php: ^8.3`. If Ubuntu 26.04 installs PHP 8.4, that is acceptable. If Ubuntu later ships PHP 8.5 directly, that is also acceptable. Avoid unsupported third-party repositories on a production server.

```bash

```

Create the production database and user. Replace `REPLACE_WITH_STRONG_DB_PASSWORD`.

```bash

```

## 9. Install Node.js LTS

The project builds Vite, React, and the Inertia SSR bundle on the server. Use a system-wide Node.js LTS install, not an `nvm`-only install. The Inertia SSR systemd service starts outside your interactive shell, so it must be able to find Node without relying on shell startup files.

As of June 2026, Node.js 24 is LTS and Node.js 22 is still LTS. Prefer Node.js 24 for a new server.

Install Node.js with NodeSource:

```bash
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo bash -
sudo apt install -y nodejs
node -v
npm -v
command -v node
```

Confirm Node is available. The SSR service uses `INERTIA_SSR_RUNTIME=node`, and the systemd service sets `PATH` so `node` resolves correctly.

```text
/usr/bin/node
/usr/local/bin/node
/usr/bin/nodejs
```

If NodeSource does not support Ubuntu 26.04 yet, try the NodeSource script first anyway. If you must use `nvm`, keep `INERTIA_SSR_RUNTIME=node` and add the `nvm` Node directory to the SSR systemd service `PATH`, for example:

```ini
Environment=PATH=/home/znar/.nvm/versions/node/v24.x.x/bin:/usr/local/bin:/usr/bin:/bin
```

A system-wide Node install is simpler and less fragile.

## 16. Configure Caddy

Create the Caddy log directory and give the Caddy service user write access:

```bash
sudo mkdir -p /var/log/caddy
sudo chown -R caddy:caddy /var/log/caddy
sudo chmod 755 /var/log/caddy
```

Edit the Caddyfile:

```bash
sudo nano /etc/caddy/Caddyfile
```

Use this config, replacing the domain if needed:

```caddyfile
goanperfume.de {
    root * /var/www/goan-perfumes/public

    encode zstd gzip

    header {
        X-Content-Type-Options nosniff
        Referrer-Policy strict-origin-when-cross-origin
        X-Frame-Options SAMEORIGIN
        Strict-Transport-Security "max-age=31536000; includeSubDomains"
        -Server
    }

    header /build/* Cache-Control "public, max-age=31536000, immutable"
    header /images/* Cache-Control "public, max-age=31536000, immutable"
    header /storage/* Cache-Control "public, max-age=31536000, immutable"

    @blocked {
        path /.env*
        path /composer.*
        path /package*.json
        path /vite.config.*
        path /artisan
    }
    respond @blocked 404

    php_fastcgi 127.0.0.1:9000
    file_server

    log {
        output file /var/log/caddy/goanperfume.access.log
        format json
    }
}

www.goanperfume.de {
    redir https://goanperfume.de{uri} permanent
}
```

Notes:

- `php_fastcgi` already performs the Laravel front-controller fallback to `index.php`, so a separate `try_files` line is unnecessary.
- `/images/*` is intentionally cached because this project serves versioned category fallback images from `public/images`.
- `/storage/*` is cached as immutable because product images and uploaded logo files use generated filenames. Do not overwrite public storage files in place; upload a new file and store a new path.
- Add `preload` to the HSTS header only after the domain, subdomains, and HTTPS behavior have been stable for a while.

Validate and reload:

```bash
sudo caddy fmt --overwrite /etc/caddy/Caddyfile
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
sudo systemctl status caddy
```

If Caddy cannot issue a certificate, check DNS, firewall, and whether another service is using ports 80 or 443:

```bash
sudo ss -tulpn | grep -E ':80|:443'
sudo journalctl -u caddy -n 100 --no-pager
```

## 17. Queue Worker

The current app does not dispatch queued jobs, so you can skip this section while `QUEUE_CONNECTION=sync`.

Only create this service later if `.env` is changed to `QUEUE_CONNECTION=database` or the app starts dispatching queued jobs.

Create a systemd service:

```bash
sudo nano /etc/systemd/system/goan-perfumes-queue.service
```

Use:

```ini
[Unit]
Description=Goan Perfume Laravel Queue Worker
After=network.target postgresql.service

[Service]
Type=simple
User=znar
Group=www-data
WorkingDirectory=/var/www/goan-perfumes
ExecStart=/usr/bin/php artisan queue:work database --sleep=3 --tries=3 --timeout=90 --max-time=3600
Restart=always
RestartSec=5
KillSignal=SIGTERM
TimeoutStopSec=90

[Install]
WantedBy=multi-user.target
```

Enable it:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now goan-perfumes-queue
sudo systemctl status goan-perfumes-queue
```

Logs:

```bash
sudo journalctl -u goan-perfumes-queue -f
```

## 18. Configure Inertia SSR

The production site should run Inertia SSR so public pages return rendered HTML, not an empty SPA shell.

Before creating or starting the service, confirm the SSR bundle exists and the runtime is available:

```bash
cd /var/www/goan-perfumes
npm run build
test -f bootstrap/ssr/app.js
node -v
command -v node
php artisan config:show inertia.ssr.runtime
```

If `bootstrap/ssr/app.js` is missing, `php artisan inertia:start-ssr` cannot serve SSR. Run `npm ci` and `npm run build` again before continuing.

Create a systemd service:

```bash
sudo nano /etc/systemd/system/goan-perfumes-ssr.service
```

Use:

```ini
[Unit]
Description=Goan Perfume Inertia SSR Server
After=network.target postgresql.service

[Service]
Type=simple
User=znar
Group=www-data
WorkingDirectory=/var/www/goan-perfumes
ExecStart=/usr/bin/php artisan inertia:start-ssr
Restart=always
RestartSec=5
KillSignal=SIGTERM
TimeoutStopSec=30
Environment=NODE_ENV=production

[Install]
```

Enable it:

```bash
sudo systemctl daemon-reload

sudo systemctl status goan-perfumes-ssr
```

Check SSR:

```bash
cd /var/www/goan-perfumes
```

If the service is active but `inertia:check-ssr` says the server is not running, check the logs:

```bash

sudo journalctl -u goan-perfumes-ssr -n 100 --no-pager
```

Logs:

```bash
sudo journalctl -u goan-perfumes-ssr -f
```


## 22. Rollback

Rollback is manual with this simple directory layout. Prefer rolling back to a known Git tag or commit:

```bash
cd /var/www/goan-perfumes
git fetch --all --tags
git checkout TAG_OR_COMMIT
composer install --no-dev --prefer-dist --optimize-autoloader --no-interaction
npm ci
npm run build
php artisan optimize
```

Restart long-running services after rollback:

```bash
PHP_VERSION="$(php -r 'echo PHP_MAJOR_VERSION.".".PHP_MINOR_VERSION;')"

sudo systemctl reload caddy
sudo systemctl restart goan-perfumes-ssr
```

## 23. Backups

Create a backup directory:

```bash
sudo mkdir -p /var/backups/goan-perfumes
sudo chmod 700 /var/backups/goan-perfumes
```

Manual database backup:

```bash
sudo -u postgres pg_dump --format=custom goan_perfumes | sudo tee /var/backups/goan-perfumes/goan_perfumes-$(date +%F-%H%M).dump > /dev/null
```

Manual uploaded files backup:

```bash
sudo tar -czf /var/backups/goan-perfumes/storage-$(date +%F-%H%M).tar.gz -C /var/www/goan-perfumes storage
```

Automate daily backups with retention:

```bash
sudo nano /usr/local/sbin/goan-perfumes-backup
```

Use:

```bash
#!/usr/bin/env bash
set -euo pipefail

BACKUP_DIR="/var/backups/goan-perfumes"
STAMP="$(date +%F-%H%M)"

mkdir -p "$BACKUP_DIR"
chmod 700 "$BACKUP_DIR"

sudo -u postgres pg_dump --format=custom goan_perfumes \
  > "$BACKUP_DIR/goan_perfumes-$STAMP.dump"

tar -czf "$BACKUP_DIR/storage-$STAMP.tar.gz" \
  -C /var/www/goan-perfumes \
  storage

find "$BACKUP_DIR" -type f -name 'goan_perfumes-*.dump' -mtime +14 -delete
find "$BACKUP_DIR" -type f -name 'storage-*.tar.gz' -mtime +14 -delete
```

Make it executable and test it:

```bash
sudo chmod 750 /usr/local/sbin/goan-perfumes-backup
sudo /usr/local/sbin/goan-perfumes-backup
sudo ls -lh /var/backups/goan-perfumes
```

Add a daily cron entry:

```bash
sudo crontab -e
```

Add:

```cron
15 2 * * * /usr/local/sbin/goan-perfumes-backup >> /var/log/goan-perfumes-backup.log 2>&1
```

Recommended backup policy:

- Daily database backup.
- Daily uploaded-files backup.
- The local automated script above keeps 14 days of backups on the server.
- Copy backups off the server to Hetzner Storage Box, S3-compatible storage, or another machine for real disaster recovery.
- Keep at least 7 daily and 4 weekly backups in the off-server backup location.
- Test restore before trusting backups.

Example restore test on a separate machine:

```bash
createdb goan_perfumes_restore
pg_restore --dbname=goan_perfumes_restore goan_perfumes-YYYY-MM-DD-HHMM.dump
```

## 24. Logs And Troubleshooting

Laravel logs:

```bash
sudo tail -f /var/www/goan-perfumes/storage/logs/laravel.log
```

Caddy logs:

```bash
sudo journalctl -u caddy -f
sudo tail -f /var/log/caddy/goanperfume.access.log
```

PHP-FPM logs:

```bash
PHP_VERSION="$(php -r 'echo PHP_MAJOR_VERSION.".".PHP_MINOR_VERSION;')"
sudo journalctl -u "php${PHP_VERSION}-fpm" -f
sudo tail -f "/var/log/php${PHP_VERSION}-fpm.log"
```

Queue logs, only if you enabled the optional queue worker:

```bash
sudo journalctl -u goan-perfumes-queue -f
```

SSR logs:

```bash
sudo journalctl -u goan-perfumes-ssr -f
```

Common fixes:

- `502 Bad Gateway`: check the PHP-FPM service, confirm it is listening on `127.0.0.1:9000`, and check Caddy logs.
- Blank initial HTML: check `goan-perfumes-ssr` and run `php artisan inertia:check-ssr`.
- Missing images: check `FILESYSTEM_DISK=public`, `php artisan storage:link`, and `storage/app/public` permissions.
- Old frontend assets: run `npm ci`, `npm run build`, reload Caddy, and clear browser cache.
- Config changes ignored: run `php artisan optimize:clear` then `php artisan optimize`.
- Queue jobs not processing: this only applies if you later switch to `QUEUE_CONNECTION=database`; then check `goan-perfumes-queue` and `failed_jobs`.
